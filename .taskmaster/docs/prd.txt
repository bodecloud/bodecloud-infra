# Product Requirements Document: bocloud.org Professional Developer Portfolio

## Executive Summary

Create a professional, minimalist portfolio website for bocloud.org that showcases technical expertise in self-hosted infrastructure, AI tools, and backend development. The primary goal is to facilitate hiring opportunities by demonstrating real-world technical capabilities without appearing desperate or overly sales-focused.

**NEW SECONDARY OBJECTIVE**: Create a comprehensive technical knowledge sharing section that provides guides, tutorials, and documentation for self-hosted infrastructure deployment, similar to guides.viren070.me but focused on the specific stack and services running in the bocloud.org infrastructure.

## Problem Statement

The current developer is a 30-year-old self-taught software engineer with ADHD who:

- Has extensive technical skills but struggles with traditional job application processes
- Runs sophisticated self-hosted infrastructure (Kubernetes, Tailscale, Oracle Cloud)
- Has built and deployed multiple AI tools and services
- Needs a professional presence that speaks to technical competence without corporate fluff
- Wants to transition from unstable gig work to stable remote employment
- Prefers text-based, async communication over phone/video interviews
- **NEW**: Has built an extensive infrastructure with 100+ services and wants to share knowledge about how others can replicate similar setups

## Target Audience

### Primary Audience

- Technical hiring managers and recruiters
- CTO/Engineering leads at startups and mid-size companies
- Freelance/contract opportunity providers
- Open source project maintainers looking for contributors

### Secondary Audience

- Developers and engineers who might collaborate or refer opportunities
- Users of the hosted tools and services
- Potential clients for automation/infrastructure consulting

### **NEW: Technical Knowledge Sharing Audience**

- Self-hosting enthusiasts and homelab operators
- Developers interested in infrastructure automation
- People looking to deploy similar stacks (Docker Compose, Kubernetes, Oracle Cloud)
- Technical community members seeking practical deployment guides
- Students and professionals learning about modern infrastructure practices

## Core Objectives

1. **Professional Credibility**: Establish technical authority through demonstrated infrastructure and tools
2. **Hiring Facilitation**: Make it easy for employers to understand capabilities and contact
3. **Portfolio Showcase**: Display real, working projects and infrastructure
4. **Authentic Representation**: Avoid corporate speak while maintaining professionalism
5. **User Utility**: Provide actual value to visitors through hosted tools and services
6. **NEW: Knowledge Sharing**: Create comprehensive guides and documentation for replicating the infrastructure setup
7. **NEW: Community Building**: Establish thought leadership in self-hosted infrastructure space
